# 使用[NextTrace](https://github.com/nxtrace/NTrace-V1)测试服务器大小包路由

## 安装NextTrace
```bash
curl nxtrace.org/nt |bash
```

## 大小包原理说明
部分服务商线路存在**小包、大包路由分离**的情况：小包走优质线路，大包走廉价线路，常规路由测试仅使用小包，无法发现线路缩水问题。
通过指定`--psize`载荷大小分别测试，对比路由跳段即可判断线路是否存在大小包分流。
- 大包：载荷1024字节
- 小包：载荷12字节
- 测试协议：TCP，端口80

## 分运营商测试命令
### 移动
#### 大包路由测试
```bash
nexttrace --tcp --psize 1024 zj-cm-v4.ip.zstaticcdn.com -p 80
```
#### 小包路由测试
```bash
nexttrace --tcp --psize 12 zj-cm-v4.ip.zstaticcdn.com -p 80
```

### 电信
#### 大包路由测试
```bash
nexttrace --tcp --psize 1024 gd-ct-v4.ip.zstaticcdn.com -p 80
```
#### 小包路由测试
```bash
nexttrace --tcp --psize 12 gd-ct-v4.ip.zstaticcdn.com -p 80
```

### 联通
#### 大包路由测试
```bash
nexttrace --tcp --psize 1024 sh-cu-v4.ip.zstaticcdn.com -p 80
```
#### 小包路由测试
```bash
nexttrace --tcp --psize 12 sh-cu-v4.ip.zstaticcdn.com -p 80
```

## 五、结果判断标准
分别执行大包、小包两条测试命令后，对比两条输出的路由节点（AS、运营商、出口线路）：
1. 路由跳段、出口线路完全一致：线路无大小包分流，正常；
2. 中途节点/国际出口线路不一致：存在大小包分流，带宽跑满时会劣化延迟、丢包。
